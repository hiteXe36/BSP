9  * @brief Dieses C Modul implementiert eine einfache synchrone Kommunikation
10  * zwischen vmapp (client) und dem Memory Manager (server).
11  * vmapp schreibt einen Auftrag in einen gemeinsamen Speicher. Der Server
12  * wartet auf den nächsten Auftrag, liest diesen aus, bearbeitet den Auftrag
13  * und bestätigt die erfolgreiche Bearbeitung dem Client.
14  * Da der Client auf diese Bestätigung wartet, ist die Kommunikation synchron.
15  * Der gemeinsame Speicher ist über einen Mutex geschützt. Analog zum Erzeuger
16  * Verbrauchen Problem werden zwei Semaphore zur Synchronisation zwischen Client
17  * und Server verwendet.
18  *
19  * Der Server ist für die Initialiserung und Freigabe der Komponenten
20  * verantwortlich.
21  *
22  ******************************************************************
23  */
